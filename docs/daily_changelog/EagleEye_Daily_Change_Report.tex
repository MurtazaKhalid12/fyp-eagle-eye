\subsection{2 May 2026 (working tree --- detailed technical delta)}
The following items reflect substantive edits present locally at report time; integrate them into the next commit to keep history aligned.

\begin{enumerate}[leftmargin=*]
  \item \textbf{Firmware --- HTTP snapshot API.} File \texttt{camera\_web\_server.h}: shared helper \texttt{eagleeye\_grab\_jpeg()} for single-frame JPEG extraction; \texttt{GET /capture} for mobile HTTP fallback; MJPEG stream on \texttt{GET /} unchanged. Include-guard typo corrected (erroneous leading character that broke preprocessing).
  \item \textbf{Firmware --- WebSocket preview.} New \texttt{eagleeye\_ws.h}: binary JPEG push over WebSockets on port~81 (Arduino library: WebSockets by Markus Sattler). Reduces per-frame HTTP overhead versus repeated \texttt{/capture} polling.
  \item \textbf{Firmware --- control flow.} \texttt{eagleeye-main.ino}: call \texttt{eagleeye\_ws\_begin()} after HTTP server start; service \texttt{eagleeye\_ws\_loop()} each iteration; while WebSocket clients are connected, AI inference is deferred (camera bandwidth). \textbf{Latch reset:} when WebSocket clients drop from one to zero, or when browser MJPEG streaming ends, \texttt{image\_sent\_this\_event}, \texttt{human\_confirmed}, and \texttt{clear\_scene\_count} reset so detections resume after closing Live Monitor (previously the latch could remain set because the AI loop did not run during preview).
  \item \textbf{Mobile --- Live Monitor.} \texttt{LiveMonitorScreen.js}: primary path \texttt{ws://<camera-ip>:81} with binary frames rendered via \texttt{expo-image}; cleartext HTTP retained for camera URLs in \texttt{app.json} (\texttt{usesCleartextTraffic}). Legacy \texttt{react-native-webview} dependency removed from \texttt{package.json}.
  \item \textbf{Mobile --- Gallery administration.} \texttt{useAlerts.js}: \texttt{deleteAllAlerts()} loads full \texttt{alerts} node, enqueues \texttt{public\_id} entries under \texttt{deletion\_requests} for Cloudinary cleanup by \texttt{bridge.py}, then removes \texttt{alerts}. \texttt{GalleryScreen.js}: confirmed \textbf{Delete all} control with loading state and modal dismissal.
\end{enumerate}

\textbf{Note:} Runtime JPEGs under \texttt{backend/captures/} may appear as untracked files during testing; exclude or gitignore per team policy. Local \texttt{secrets.h} changes (e.g.\ MQTT broker IP for hotspot vs.\ LAN) are not versioned by design.
